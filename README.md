# BizopdAssignment

A REST API for managing production orders through a linear stage pipeline. Built with **Node.js**, **TypeScript**, **Express**, **PostgreSQL** (via Prisma), and **Redis**.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Runtime | Node.js v22 |
| Language | TypeScript |
| Framework | Express 5 |
| Database | PostgreSQL + Prisma ORM |
| Cache | Redis |
| Validation | Zod |
| Dev Server | tsx --watch |

---

## Prerequisites

Make sure the following are installed and running on your machine before starting:

- [Node.js](https://nodejs.org/) v18+
- [PostgreSQL](https://www.postgresql.org/) — running on `localhost:5432`
- [Redis](https://redis.io/) — running on `localhost:6379`

Verify Redis is running:
```bash
redis-cli ping
# Expected output: PONG
```

---

## Setup

### 1. Clone the repository

```bash
git clone <repo-url>
cd BizOPD/backend
```

### 2. Install dependencies

```bash
npm install
```

### 3. Configure environment variables

Create a `.env` file in the project root:

```env
PORT=3000
DATABASE_URL="postgresql://postgres:<your-password>@localhost:5432/bizopd"
REDIS_URL="redis://localhost:6379"
```

> Replace `<your-password>` with your PostgreSQL password.

### 4. Generate Prisma client & run migrations

```bash
# Apply migrations to create the database schema
npx prisma migrate dev

# Generate the Prisma client
npx prisma generate
```

### 5. Start the development server

```bash
npm run dev
```

You should see:
```
Redis is connected
Server is running on http://localhost:3000
```

## Order Stage Pipeline

Orders move through the following stages **in order** — skipping or going backwards is not allowed:

```
PENDING_DESIGN → AWAITING_APPROVAL → PRINTING → FRAMING → PACKAGING → READY_FOR_PICKUP → DELIVERED
```

---

## API Reference

Base URL: `http://localhost:3000/api`

---

### `POST /api/orders` — Create an order

> Requires `Idempotency-Key` header to prevent duplicate submissions.

**Headers:**
```
Content-Type: application/json
Idempotency-Key: <unique-uuid>
```

**Request Body:**
```json
{
  "customerName": "Seyi Thatti",
  "phone": "09044281985",
  "product": "Black frame 20x30",
  "stage": "PENDING_DESIGN"
}
```
> `stage` is optional — defaults to `PENDING_DESIGN`.

**Success Response `201`:**
```json
{
  "id": "a1b2c3d4-...",
  "customerName": "Seyi Thatti",
  "phone": "09044281985",
  "product": "Black frame 20x30",
  "stage": "PENDING_DESIGN",
  "createdAt": "2026-06-21T01:00:00.000Z",
  "updatedAt": "2026-06-21T01:00:00.000Z"
}
```

---

### `PATCH /api/orders/:id/stage` — Advance an order's stage

> Requires `Idempotency-Key` header. Only moves to the **next** stage in the pipeline.

**Headers:**
```
Content-Type: application/json
Idempotency-Key: <unique-uuid>
```

**Request Body:**
```json
{
  "newStage": "AWAITING_APPROVAL"
}
```

**Success Response `200`:** Returns the updated order object.

**Error Responses:**
| Status | Reason |
|---|---|
| `404` | Order ID not found |
| `422` | Invalid stage transition (skipping, going backwards, or already DELIVERED) |

---

### `GET /api/orders` — List all orders

**Query Params (optional):**

| Param | Description |
|---|---|
| `search` | Filter by customer name or phone number (case-insensitive) |

**Examples:**
```
GET /api/orders
GET /api/orders?search=Seyi
GET /api/orders?search=09044
```

**Success Response `200`:**
```json
[
  {
    "id": "a1b2c3d4-...",
    "customerName": "Seyi Thatti",
    "phone": "09044281985",
    "product": "Black frame 20x30",
    "stage": "PENDING_DESIGN",
    "createdAt": "2026-06-21T01:00:00.000Z",
    "updatedAt": "2026-06-21T01:00:00.000Z"
  }
]
```

---

### `GET /api/dashboard` — Dashboard stats

Returns aggregate counts. Response is **cached in Redis for 60 seconds**.

**Success Response `200`:**
```json
{
  "totalOrders": 12,
  "inProduction": 9,
  "delivered": 3
}
```

## Testing the APIs (curl)

### Create an order
```bash
curl -X POST http://localhost:3000/api/orders \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: $(uuidgen)" \
  -d '{
    "customerName": "Seyi Thatti",
    "phone": "09044281985",
    "product": "Black frame 20x30"
  }'
```

### Advance the order to the next stage
```bash
curl -X PATCH http://localhost:3000/api/orders/<order-id>/stage \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: $(uuidgen)" \
  -d '{ "newStage": "AWAITING_APPROVAL" }'
```

### Search orders
```bash
curl http://localhost:3000/api/orders?search=Seyi
```

### Get dashboard stats
```bash
curl http://localhost:3000/api/dashboard
```

## Idempotency

All mutating endpoints (`POST`, `PATCH`) require an `Idempotency-Key` header — a unique UUID generated by the client per request.

- **First request:** processed normally, response cached in Redis for **24 hours**
- **Retry with same key:** cached response returned immediately, no DB write
- **Missing key:** `400 Bad Request`
- **Failed requests (5xx):** not cached, safe to retry with the same key

```bash
# Generate a key once, reuse it on retries
KEY=$(uuidgen)

curl -X POST http://localhost:3000/api/orders \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: $KEY" \
  -d '{ "customerName": "Seyi", "phone": "09044281985", "product": "frame" }'

# Retry — will get the same response, no duplicate order created
curl -X POST http://localhost:3000/api/orders \
  -H "Content-Type: application/json" \
  -H "Idempotency-Key: $KEY" \
  -d '{ "customerName": "Seyi", "phone": "09044281985", "product": "frame" }'
```
